Course Title: Agent Skills with Anthropic
Course Link: https://www.deeplearning.ai/courses/agent-skills-with-anthropic
Course Instructor: Elie Schoppik

Lesson 1: Introduction
Agent Skills with Anthropic teaches how to create reusable skills that extend the capabilities of AI agents. Skills are folders of instructions that transform general-purpose agents into specialists. This course covers the structure of skills, best practices for writing them, and how to integrate them with Claude.ai, Claude Code, the Claude API, and the Claude Agent SDK. You will learn how to combine skills with MCP servers and subagents to build complex, composable AI systems. By the end of this course, you will be able to create custom skills for any domain, share them across projects and teams, and integrate them into production AI pipelines.

Lesson 2: Why Use Skills - Part I
Skills solve a fundamental problem with general-purpose AI agents: they are broad but not deep. A general agent knows a little about everything but lacks the specialized knowledge and workflow expertise needed for domain-specific tasks. Skills are reusable instruction sets that give agents deep expertise in a specific area. A skill is a directory containing a markdown file with instructions, examples, context, and optionally tool definitions and sub-workflows. When an agent loads a skill, it gains the knowledge and behavior defined in that skill for the duration of the task. Why skills matter: they make agent behavior consistent and reproducible, they encode expert knowledge that would otherwise require extensive prompting, they can be versioned and shared across teams, and they reduce the need to repeat context in every prompt. Skills work by providing structured context to the agent before it begins a task. This context includes what the skill does, when to use it, step-by-step instructions, examples of correct behavior, and common pitfalls to avoid. The agent uses this context to specialize its behavior for the task at hand.

Lesson 3: Why Use Skills - Part II
Skills enable a new model of AI development where capabilities are modular and composable. Instead of building one large monolithic agent, you build a collection of skills that can be combined as needed. This modularity has several advantages: skills can be tested and improved independently, teams can develop skills in parallel, skills can be reused across different agents and projects, and new capabilities can be added by creating new skills without modifying existing ones. Skills also solve the context management problem. Instead of stuffing all instructions into a system prompt, skills are loaded on demand, keeping the active context focused and efficient. A skill is triggered by a user request or by another agent recognizing that the skill is relevant. The skill loader pattern: an orchestrator agent receives a task, identifies which skills are needed, loads those skills into the context, and delegates execution to a specialized sub-agent. This pattern enables multi-agent systems where each agent is an expert in its domain. Skills can also include guardrails: instructions that tell the agent what NOT to do, edge cases to watch out for, and how to handle errors or ambiguous situations.

Lesson 4: Skills vs Tools, MCP, and Subagents
Understanding the distinction between skills, tools, MCP servers, and subagents is essential for designing effective AI systems. Tools are functions that an agent can call to interact with the world: reading files, searching the web, running code, calling APIs. Tools provide capabilities. Skills provide knowledge and instructions about how to use those capabilities effectively. MCP (Model Context Protocol) servers expose tools to agents in a standardized way. An MCP server can provide file system access, database queries, web browsing, or any external API. Skills and MCP are complementary: a skill might instruct the agent on how to use a specific MCP server effectively. Subagents are separate agent instances that handle specific subtasks. The difference between a skill and a subagent: a skill augments a single agent's behavior, while a subagent is an independent agent that runs in parallel or in sequence. Skills are loaded into the current agent's context. Subagents receive their own context and run independently. When to use each: use skills when you need to add domain knowledge or workflow instructions to an agent, use MCP when you need to give an agent access to external systems, use subagents when you need parallel execution or want to isolate complex subtasks. These three mechanisms are designed to work together: a skill might instruct an agent to spin up a subagent using a specific MCP server.

Lesson 5: Exploring Pre-Built Skills
Anthropic provides a library of pre-built skills that cover common development workflows. Exploring these skills is the best way to understand the structure and conventions for writing your own. Pre-built skills include: code-review (reviews code for bugs, style issues, and improvements), security-review (performs security analysis of code changes), implement-feature (guides the agent through feature implementation with planning and testing), run (starts the application and verifies behavior), verify (validates that a change works correctly), and init (initializes a CLAUDE.md file for a new project). Each skill follows a consistent structure: a frontmatter section with the skill name and description, a trigger section that defines when the skill should be used, a step-by-step instructions section, examples of correct behavior, and notes on edge cases and limitations. Reading pre-built skills reveals important patterns: skills use imperative language ("do X before Y"), they anticipate common mistakes, they provide decision trees for handling different scenarios, and they reference other skills when relevant. The skills directory is typically located at ~/.claude/skills/ for global skills or .claude/skills/ for project-specific skills. Skills are loaded automatically when their trigger conditions are met, or explicitly when the user invokes them with a slash command.

Lesson 6: Creating Custom Skills
Creating a custom skill involves writing a markdown file that encodes expert knowledge and workflow instructions for a specific domain. The anatomy of a skill file: the filename should be descriptive (e.g., deploy-to-production.md), the file starts with frontmatter defining the skill name and description, the body contains structured instructions in sections. Key sections of a skill: WHEN TO USE (trigger conditions), CONTEXT (background knowledge the agent needs), STEPS (the workflow to follow, numbered), EXAMPLES (concrete examples of correct behavior), PITFALLS (common mistakes to avoid), and OUTPUT FORMAT (what the agent should produce). Best practices for writing skills: be explicit about ordering constraints (what must happen before what), include decision points with clear criteria, provide examples of both correct and incorrect behavior, define success criteria so the agent knows when the task is complete, keep skills focused on a single domain or workflow. A skill for deploying a FastAPI application might include: steps for running tests, building a Docker image, pushing to a registry, updating environment variables, and verifying the deployment is healthy. The skill would also include rollback instructions in case of failure. Skills should be tested by running them on representative tasks and checking that the agent follows the workflow correctly. Iterate on the skill based on observed failures.

Lesson 7: Skills with the Claude API
Skills can be integrated into Claude API applications to give Claude specialized behavior for specific tasks. When using the Claude API, you can implement skills by including the skill content in the system prompt, by loading skill files dynamically based on the user's request, or by using a skill router that selects the appropriate skill for each request. Implementation pattern using the Claude API: create a skill loader that reads skill files from a directory, implement a router that maps user intents to skills, prepend the selected skill content to the system prompt before each API call. Example with the Anthropic Python SDK: load skill content from a markdown file, pass it as the system prompt to the messages.create endpoint, combine with tool definitions if the skill requires tool use. Skills with tool use: a skill can define which tools the agent should use and how to use them. For example, a data analysis skill might instruct the agent to always use the code execution tool to verify calculations, and to produce charts using matplotlib. Caching skill content: since skills are static text, they are excellent candidates for prompt caching with the Anthropic API. Use the cache_control parameter with type ephemeral to cache skill content and reduce costs on repeated calls. Multi-skill routing: when a user request could match multiple skills, implement a classification step that selects the most relevant skill, or combine multiple skills into a single context if the task requires multiple domains of expertise.

Lesson 8: Skills with Claude Code
Skills integrate natively with Claude Code as slash commands. A skill file placed in the .claude/skills/ directory of a project (or in ~/.claude/skills/ for global skills) becomes available as a slash command with the skill's filename as the command name. Example: a skill file named deploy.md becomes available as /deploy in Claude Code. When the user types /deploy, Claude Code loads the skill content and uses it to guide the deployment workflow. Project-specific vs global skills: project skills are stored in .claude/skills/ and are only available when working in that project. Global skills are stored in ~/.claude/skills/ and are available in all projects. Skills can also be triggered automatically based on context, not just by explicit slash commands, if the skill's frontmatter defines trigger conditions. Combining skills with hooks: Claude Code hooks can trigger skills automatically. For example, a PostToolUse hook could invoke a code-review skill after every file edit, or a Stop hook could trigger a summary skill when a long task completes. Skills with subagents in Claude Code: a skill can instruct Claude Code to spawn subagents using the Agent tool. The skill defines the task for the subagent, which tools it should use, and how to integrate the subagent's output into the main workflow. This enables parallel execution of independent tasks within a single Claude Code session. Best practices for Claude Code skills: keep skill files focused and concise, use CLAUDE.md to document which skills are available in the project, include examples of when to use each skill.

Lesson 9: Skills with the Claude Agent SDK
The Claude Agent SDK provides a programmatic interface for building multi-agent systems where skills are first-class citizens. The SDK allows you to define agents with specific skills, compose agents into pipelines, and orchestrate complex workflows. Core concepts of the Claude Agent SDK: Agent (an LLM with a system prompt, tools, and skills), Task (a unit of work assigned to an agent), Skill (a reusable instruction set loaded into an agent's context), Pipeline (a sequence or graph of agents and tasks). Defining an agent with skills using the SDK: specify the model, provide a base system prompt, list the skills to load, define the tools available to the agent. The SDK handles skill loading, context management, and tool execution automatically. Multi-agent patterns with the SDK: the Orchestrator-Worker pattern (one orchestrator agent routes tasks to specialized worker agents, each loaded with relevant skills), the Pipeline pattern (agents are chained in sequence, each processing the output of the previous agent), the Parallel pattern (multiple agents run simultaneously on independent subtasks, results are merged by a reducer agent). Skill composition: the SDK supports loading multiple skills into a single agent, and skills can reference other skills. This enables skill hierarchies where a high-level skill delegates to lower-level skills. Error handling in multi-agent systems: the SDK provides mechanisms for agents to signal failure, retry tasks with different skills, and escalate to human oversight when confidence is low. Best practices for the Agent SDK: design skills to be stateless so they can be reused across agents, use structured output (JSON schemas) for inter-agent communication, implement observability with traces and metrics from the start.

Lesson 10: Conclusion
Skills represent a paradigm shift in how AI capabilities are built and shared. Instead of encoding all knowledge in model weights or system prompts, skills provide a modular, versioned, and composable approach to agent specialization. Key takeaways from this course: skills make agents consistent and reproducible by encoding expert knowledge in structured markdown files, skills integrate with Claude.ai, Claude Code, the Claude API, and the Claude Agent SDK through a unified convention, skills are complementary to tools and MCP servers — tools provide capabilities, skills provide knowledge about how to use them, multi-agent systems built with skills are more maintainable than monolithic agents because each skill can be tested and improved independently, prompt caching with skills reduces API costs significantly when skills are reused across many requests. The future of agent development is compositional: teams will maintain libraries of skills that can be mixed and matched to build specialized agents for any domain. Contributing to shared skill libraries — within a team or the broader community — accelerates AI development across projects. Start by identifying repetitive prompting patterns in your current workflows, extract them into skills, and iterate based on observed agent behavior.
