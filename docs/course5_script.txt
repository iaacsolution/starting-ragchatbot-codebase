Lesson 1: Introduction
Claude Code is a highly agentic coding assistant built by Anthropic. Unlike traditional code completion tools, Claude Code can plan, execute, and improve code with minimal human input over extended periods. It operates directly in your terminal and has access to your entire codebase, allowing it to read files, run commands, write and edit code, search the web, and interact with external tools via MCP servers. In this course, you will learn best practices for using Claude Code effectively across three real-world projects: exploring a RAG chatbot codebase, refactoring a Jupyter notebook for e-commerce data analysis, and building a web app from a Figma mockup.

Lesson 2: What is Claude Code?
Claude Code is an agentic AI coding tool that runs in your terminal. It is powered by Claude, Anthropic's AI model, and is designed to work on long-horizon tasks that require multiple steps, tool use, and autonomous decision-making. Unlike copilot-style tools that suggest the next line of code, Claude Code can handle entire workflows: understanding a codebase, planning changes, writing code, running tests, debugging errors, and iterating until the task is complete. Claude Code uses a ReAct-style loop where it reasons about what to do, takes an action (read a file, run a command, write code), observes the result, and continues until the task is done. Key capabilities include: reading and writing files, executing shell commands, searching the web, using MCP servers to connect to external tools like Playwright or Figma, maintaining memory across sessions via CLAUDE.md files, and supporting hooks to automate workflows triggered by specific events. Claude Code is installed via npm: `npm install -g @anthropic-ai/claude-code`. It is invoked from your terminal with the `claude` command. You can also use slash commands like /help, /clear, /config, /review, and /compact to control behavior. The model supports both interactive and non-interactive (headless) modes for CI/CD integration.

Lesson 3: Setup and Codebase Understanding
When starting with a new codebase, providing Claude Code with the right context is the most important step. The recommended approach is to create a CLAUDE.md file at the root of your project. This file acts as persistent instructions that Claude Code reads at the start of every session. A good CLAUDE.md includes: the project description and purpose, key commands for running, testing, and building the project, the architecture overview with key file paths, non-obvious constraints and conventions, and any coding standards or patterns to follow. To explore a codebase, you can ask Claude Code to: summarize the project structure, explain the purpose of specific files or modules, trace the execution flow of a request end-to-end, identify dependencies and their roles. When exploring the RAG chatbot codebase used in this course, Claude Code can map out the flow from the FastAPI endpoint through the RAG system, vector store, and AI generator. It understands that ChromaDB is used for vector storage, that the system uses Anthropic Claude for generation, and that RAGAS is used for evaluation. Best practice: always start a new task by giving Claude Code the relevant file paths and a clear description of what you want to achieve. The more specific context you provide, the better the results.

Lesson 4: Adding Features
Claude Code excels at adding new features to existing codebases because it can read the existing code, understand patterns, and implement consistent changes across multiple files. When asking Claude Code to add a feature, the best approach is to: describe the feature clearly in terms of user-facing behavior, specify which files are relevant, reference similar existing patterns in the codebase to follow. For example, to add a streaming endpoint to the RAG chatbot, you would tell Claude Code: the desired behavior (token-by-token streaming via SSE), the file to modify (backend/app.py), and the existing patterns to follow (the /api/query endpoint structure). Claude Code will implement the feature, update relevant imports, and modify the frontend if needed. Key tips for adding features: use imperative language ("add", "implement", "create"), reference existing code patterns explicitly, ask Claude Code to explain its plan before implementing if the task is complex, break large features into smaller sub-tasks. Claude Code maintains context within a session, so you can have a conversation: ask it to implement something, review the result, ask for adjustments, and iterate.

Lesson 5: Testing, Error Debugging and Code Refactoring
Claude Code is highly effective at writing tests, debugging errors, and refactoring code. For testing: ask Claude Code to write unit tests for a specific function or module, specify the testing framework (pytest, jest, etc.), and tell it to use mocks for external dependencies (APIs, databases). Claude Code can also run the tests and iterate on failures automatically. For debugging: paste the error message and stack trace directly into Claude Code, provide the relevant file paths, and ask it to diagnose and fix the issue. Claude Code will read the code, identify the root cause, implement a fix, and verify it resolves the error. For refactoring: describe the goal (improve readability, extract a class, reduce duplication), specify the files involved, and ask Claude Code to maintain existing behavior while improving structure. Claude Code follows the principle of behavior-preserving refactoring: it runs existing tests before and after to confirm nothing broke. Best practices: when debugging, always include the full error message and the command used to trigger it. When refactoring, ask Claude Code to explain what it changed and why. Use /review to get a code review of recent changes.

Lesson 6: Adding Multiple Features Simultaneously
One of Claude Code's most powerful capabilities is handling complex, multi-step tasks that span multiple files and components. When adding multiple features at once, provide Claude Code with a comprehensive task description that lists all the changes needed. Claude Code will plan the work, implement changes in a logical order, and handle dependencies between features automatically. For example, adding streaming support to the RAG chatbot requires changes in: the backend (new endpoint, async generator), the AI generator (async streaming method), and the frontend (SSE consumer, UI update logic). Claude Code handles all of these in a single session. Tips for multi-feature tasks: use a numbered list to specify all required changes, mention any ordering constraints (e.g., "implement the backend endpoint before updating the frontend"), ask Claude Code to create a plan first before implementing. Claude Code tracks its progress and can resume interrupted tasks. It uses git status and git diff to understand what has been done and what remains.

Lesson 7: Exploring GitHub Integration and Hooks
Claude Code integrates deeply with git and GitHub workflows. Git operations: Claude Code can run git commands, create commits with meaningful messages, create branches, and open pull requests using the GitHub CLI (gh). It follows best practices: staging specific files rather than git add -a, writing descriptive commit messages, never force-pushing to main. Hooks are a powerful Claude Code feature that lets you automate actions triggered by specific events in Claude Code's lifecycle. Hook events include: PreToolUse (before a tool runs), PostToolUse (after a tool runs), Stop (when Claude Code finishes a turn), and Notification (when Claude Code sends a notification). Common hook use cases: auto-format code with Black or Prettier after every file edit, run tests after every Python file change, lint with Ruff after edits, play a sound or send a notification when a long task completes. Hooks are configured in settings.json under the hooks key. Each hook specifies: a matcher (which tool triggers it, e.g., "Edit|Write"), a command to run, and a statusMessage shown to the user. Example hook to run Black after editing Python files: the command reads the tool input from stdin, extracts the file path, and runs `uv run black <file>` if the file ends in .py. Hooks receive the tool input as JSON via stdin, making them highly flexible. CLAUDE.md files can document project-specific hooks so that all team members use the same automation.

Lesson 8: Refactoring a Jupyter Notebook and Creating a Dashboard
Claude Code can work with Jupyter notebooks to refactor analysis code and create interactive dashboards. When working with notebooks, Claude Code can: read cell contents and outputs, identify repeated patterns across cells, extract reusable functions, restructure the analysis flow for clarity, and add visualizations. For e-commerce data analysis, Claude Code can refactor a notebook that loads sales data, cleans it, and produces ad-hoc charts into a clean, modular pipeline with reusable functions and a Plotly or Streamlit dashboard. Best practices for notebooks: tell Claude Code the goal of the notebook (exploratory analysis, production pipeline, stakeholder dashboard), specify the libraries already in use, ask it to preserve existing visualizations while improving structure. Claude Code can also install missing dependencies by running pip install or uv add commands and verify the notebook runs end-to-end after refactoring.

Lesson 9: Creating a Web App Based on a Figma Mockup
Claude Code can connect to Figma via the Figma MCP server to read design files and implement them as web applications. The Figma MCP server provides Claude Code with access to design components, layouts, colors, typography, and assets directly from Figma. Workflow: connect the Figma MCP server in Claude Code settings, share the Figma file URL with Claude Code, ask it to implement the design as HTML/CSS/JavaScript or in your preferred framework. Claude Code will read the Figma design, extract component specifications, and generate code that matches the visual design. It handles responsive layouts, color variables, typography scales, and component states. Tips for Figma-to-code tasks: start with the overall layout before diving into components, specify the target framework (plain HTML, React, Vue, etc.), ask Claude Code to use CSS variables for colors and spacing to match the design system, review the output in a browser and ask for adjustments iteratively. Claude Code can also use the Playwright MCP server to take screenshots of the implemented app and compare them to the Figma mockup, enabling visual regression testing during development.

Lesson 10: Conclusion
Claude Code represents a new paradigm in software development: agentic AI that can handle long-horizon coding tasks with minimal human supervision. The key to using Claude Code effectively is providing clear context (via CLAUDE.md), specifying relevant files and patterns, and iterating collaboratively. Best practices summary: always maintain a CLAUDE.md file with project context and conventions, use hooks to automate repetitive tasks like formatting, linting, and testing, connect MCP servers to extend Claude Code's capabilities (Playwright, Figma, databases), break complex tasks into clear sub-tasks with explicit success criteria, use /review to validate changes before committing, leverage git integration for clean commit history and pull requests. Claude Code is most powerful when treated as a collaborative partner: give it context, review its work, provide feedback, and iterate. The combination of autonomous execution and human oversight is what makes agentic coding productive and safe.
